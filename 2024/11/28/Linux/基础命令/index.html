1.10 分页查看文件：less,more

more 和 less 是两个看文件内容的命令，功能类似，但有些区别。

more：一页一页地看

  • 基本语法：

  • 特点：

    • 每次显示一屏内容。
    • 按 Enter 往下滚一行，按 空格 翻一页。
    • 到文件末尾后退出。
    • 按 q 退出。

  • 不足：

    • 不能往回翻（对，没办法后悔）。
    • 功能少，交互能力弱。
less：牛逼的文件浏览器

  • 基本语法：

  • 特点：

    • 支持往回翻，用 ↑ 和 ↓ 上下滚动，或者直接翻页：
      • PageUp：上一页
      • PageDown：下一页
    • 查找内容：
      • 输入 /关键字 按 Enter 查找。
      • 按 n 找下一个匹配，N 找上一个。
    • 文件再长也能搞定，不会一次性读入整个文件（节省内存）。
    • 按 q 退出。
对比
功能 more less
是否可往回翻 否 是
查找功能 没有 有
性能 简单文件能顶 适合大文件，效率高

1.11 which,find

  • 找程序用 which。
  • 全盘搜文件用 find
1. which：找程序路径
作用：
  • 查看某个命令对应的可执行文件路径。
基本用法：

  1. 查单个命令路径：

    结果：显示 ls 命令的绝对路径，比如 /bin/ls。

  2. 查多个命令路径：

    结果：分别显示 ls 和 grep 命令的路径。

常用选项：

  • -a：显示所有匹配的路径（包括环境变量中同名的多个可执行文件）。

    结果：可能返回多个 python 版本路径。

局限性：
  • 只找可执行程序，不管普通文件或目录。
  • 仅限搜索环境变量 $PATH 中的目录，其他路径找不到。
2. find：全能搜索器
作用：
  • 在指定目录及其子目录中搜索文件或目录。
基本用法：

  1. 按文件名搜索：

    示例：

    搜索 /home 目录下所有 .txt 文件。

  2. 搜索目录：

    搜索名为 test 的目录。

  3. 按大小搜索：

    找出大于 10MB 的文件。

  4. 按权限搜索：

    找出权限为 755 的文件。

常用选项：

  • -type：指定类型（f 文件，d 目录）。

  • -iname：忽略大小写匹配文件名。

  • -exec：对搜索结果执行操作。

    找出当前目录的 .log 文件并删除。

优点：
  • 搜索范围不限，能找任何文件或目录。
  • 支持强大的筛选条件（按时间、大小、权限等）。
对比总结：
功能 which find
搜索范围 只在 $PATH 中的可执行文件 任意目录和文件
搜索对象 可执行程序 文件、目录等
是否递归 不递归 递归搜索
功能复杂度 简单，专注找程序 强大，可按多种条件筛选
常用场景 查看命令路径、定位程序安装位置 全盘找文件、目录、或按属性搜索
