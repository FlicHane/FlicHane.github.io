allow-circular-references: true
是 Spring Framework 中的一个配置选项，用于控制 循环依赖（Circular Dependencies）的处理。

什么是循环依赖？

循环依赖指的是两个或多个 Bean 互相引用，形成一个闭环，导致容器无法正常实例化它们。例如：

allow-circular-references: true 配置的作用

在 Spring 5 和以前的版本中，Spring 容器默认允许存在 单例 Bean 的循环依赖，即 Spring 会使用 三级缓存（两次实例化和一次填充依赖）来解决循环依赖的问题。

但是在 Spring 6 及更高版本中，默认不再允许循环依赖，需要显式地配置 allow-circular-references: true 才能重新启用循环依赖的处理。

如果启用 allow-circular-references: true，Spring 会尝试通过 延迟初始化（延迟装配）和 代理（例如使用 JDK 动态代理或 CGLIB 代理）来解决循环依赖。

什么时候需要使用它？

  1. 存在明显的循环依赖时：
    当你的应用中确实有不可避免的循环依赖时，可以启用此配置来允许 Spring 自动处理这些依赖关系。

  2. 开发阶段，避免重构代码时：
    如果你正在开发一个较大的项目，临时启用此选项避免代码重构或调整大量 Bean 定义。

注意事项：

  • 避免滥用： 启用循环依赖虽然能让 Spring 处理这类情况，但这通常是设计不良的信号。尽量避免在设计中出现循环依赖，因为它会使代码变得复杂和难以维护。

  • 使用构造器注入： 如果你使用构造器注入来管理依赖关系，通常能避免循环依赖，因为构造器注入要求所有依赖在对象创建时就已提供。

总结：

allow-circular-references: true 配置是让 Spring 容器允许并处理单例 Bean 之间的循环依赖问题。在默认情况下（特别是 Spring 6 以后），如果没有显式配置此选项，Spring 会拒绝循环依赖，因此你需要显式启用该选项来允许其处理循环依赖。
